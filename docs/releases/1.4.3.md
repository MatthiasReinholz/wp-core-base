# wp-core-base 1.4.3

## Summary

- Added retry-aware premium metadata fetch behavior by routing premium `requestJson('GET', ...)` calls through the HTTP client's JSON retry path.
- Added premium-provider tuning hooks for metadata requests:
  - `premiumMetadataTimeoutSeconds()`
  - `premiumMetadataRetryAttempts()`
  - `premiumMetadataInitialRetryDelayMilliseconds()`
- Added per-request HTTP option support for timeout and retry controls in the shared HTTP client internals.
- Updated premium provider scaffolding and provider documentation to explain metadata retry/timeout tuning.

## Downstream Impact

- Downstream repositories using custom premium providers gain safer handling of transient provider timeouts and intermittent upstream failures.
- Existing providers remain backward-compatible without code changes.
- Providers that need stronger resiliency can opt in by overriding the new metadata tuning hooks.

## Migration Notes

- No mandatory migration is required.
- Optional: for providers with unstable metadata endpoints, override metadata timeout/retry hooks in the downstream provider class.

## Downstream Workflow Changes

- None required.

## Required Downstream Actions

1. Review custom premium provider implementations.
2. If provider metadata endpoints are latency-prone, override metadata timeout/retry hook methods.
3. Re-run `doctor --automation --json` after provider adjustments.

## Bundled Baseline

- WordPress baseline unchanged from 1.4.2.
