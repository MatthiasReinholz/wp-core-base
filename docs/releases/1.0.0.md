# wp-core-base 1.0.0

## Summary

This is the first formal SemVer release of `wp-core-base` as a versioned framework product.

It establishes the framework metadata model, the gated release flow, and vendored framework self-update support for downstream repositories.

## Downstream Impact

Downstream repositories can now pin an installed `wp-core-base` framework version in `.wp-core-base/framework.php` and receive dedicated PRs when a newer framework release is available.

The framework remains compatible with the existing manifest-driven runtime model and continues to support both `full-core` and `content-only` downstreams.

## Migration Notes

Existing downstreams should scaffold or add `.wp-core-base/framework.php` and the `wp-core-base` self-update workflow if they want automated framework update PRs.

Framework updates are optimized for vendored snapshots under `vendor/wp-core-base`. Teams that rely on other install modes can still adopt tagged releases manually.

## Bundled Baseline

- WordPress core: `6.9.4`
- Managed bundled plugins:
  - WooCommerce `10.6.1`
  - Jetpack `15.6`
  - Contact Form 7 `6.1.5`
  - Redirection `5.7.5`

## Operational Notes

The release asset published for this version is the vendorable framework snapshot `wp-core-base-vendor-snapshot.zip`.
